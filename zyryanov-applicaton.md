## [ НАЗВАНИЕ ПРОЕКТА ]
Steam Analysis
## [ ПРОБЛЕМНОЕ ПОЛЕ ]
- разработчики получают тысячи текстовых отзывов без структурированной аналитики
- сложно быстро понять, какие аспекты игры критикуют, а какие хвалят
- Steam показывает только общий рейтинг, не объясняет причины
- ручной анализ отзывов требует много времени и ресурсов
- существующие инструменты слабо ориентированы на нужды разработчиков

## [ ИНСТРУМЕНТЫ РАЗРАБОТКИ, ИНФОРМАЦИЯ О БД ]
### Данные
- готовый датасет kaggle Steam Reviews за 2024 год
- Steam Works Reviews API для инкрементального обновления отзывов
- открытые списки популярных игр для добавления новых проектов

### Backend
- язык программирования Python
- web-фреймворк FastAPI для реализации серверной логики
- SQLite
- библиотека spaCy для токенизации и разбиения текста на предложения
- библиотека sentence-transformers для получения векторных представлений текста
- предобученная модель sentence-transformers (all-MiniLM-L6-v2) для семантического анализа предложений
- библиотека scikit-learn для вспомогательных алгоритмов

### Frontend
- HTML для структуры пользовательского интерфейса
- CSS для оформления и визуального разделения данных
- минимальный JavaScript для взаимодействия с backend API
- веб-браузер для отображения интерфейса и работы пользователя

### Общие инструменты
- система контроля версий Git
- репозиторий GitHub для хранения кода и документации
